# 🍔 Burger Builder

Интерактивное веб-приложение "Построй свой бургер" с полным стеком технологий: Node.js/TypeScript бэкенд, PostgreSQL база данных и современный React фронтенд.

## ✨ Основные возможности {#features}

- 🎨 **Интерактивный конструктор бургеров** с drag-and-drop функциональностью
- 🦄 **Шуточные ингредиенты** (Котлета из единорога, Сыр с планеты Марс, и др.)
- 🎭 **Визуализация сборки бургера** с анимациями слой за слоем
- 💾 **Сохранение рецептов** в PostgreSQL базе данных
- 🖼️ **Галерея созданных бургеров** с рейтингами
- 🚩 **Feature Flags система** для управления функциональностью
- 📱 **Responsive дизайн** для мобильных устройств
- 🎪 **Easter eggs** и микро-анимации

## 🏗️ Архитектура системы {#architecture}

### Фронтенд

- **Фреймворк**: React с TypeScript
- **Сборщик**: Vite для разработки и продакшена
- **Стили**: Tailwind CSS с кастомной цветовой палитрой в бургерной тематике
- **UI компоненты**: Radix UI с стилизацией shadcn/ui
- **Управление состоянием**: React Query (TanStack Query) для серверного состояния
- **Роутинг**: Wouter для легковесной маршрутизации
- **Анимации**: Framer Motion для интерактивных эффектов

### Бэкенд

- **Платформа**: Node.js с TypeScript
- **Фреймворк**: Express.js для REST API
- **ORM**: Drizzle ORM с PostgreSQL
- **Драйвер БД**: Neon serverless PostgreSQL клиент
- **Сессии**: PostgreSQL-based сессии через connect-pg-simple
- **Обработка ошибок**: Централизованная обработка с user-friendly страницами ошибок

## 🗄️ Структура базы данных {#database}

### Таблицы (shared/schema.ts)

- **burgers**: Хранение рецептов бургеров с ингредиентами, калориями, сложностью
- **ratings**: Пользовательские рейтинги и комментарии (зависит от feature flags)
- **ingredients**: Мастер-список доступных ингредиентов по категориям

## 🛠️ API Endpoints {#api}

### Основные маршруты (server/)

- **GET /api/burgers**: Получение всех сохраненных бургеров
- **POST /api/burgers**: Сохранение нового рецепта
- **GET /api/ingredients**: Получение списка доступных ингредиентов
- **GET /api/version**: Информация о версии и feature flags
- **Рейтинговые endpoints**: Условно активные в зависимости от feature flags

## ⚛️ Ключевые компоненты {#components}

### Фронтенд компоненты (client/src/components/)

- **BurgerBuilder**: Основной интерфейс интерактивной сборки бургера
- **IngredientSidebar**: Выбор ингредиентов по категориям с поддержкой drag-and-drop
- **BurgerGallery**: Отображение сохраненных бургеров с рейтингами
- **BurgerStats**: Расчет статистики и шуточных метрик

### Кастомные хуки

- **useBurgerBuilder**: Управление состоянием бургера, расчетами и сохранением
- **useToast**: Система уведомлений
- **useMobile**: Утилиты для адаптивного дизайна

## 📊 Поток данных {#data-flow}

1. **Загрузка ингредиентов**: Приложение получает список ингредиентов из `/api/ingredients` при запуске
2. **Сборка бургера**: Пользователь взаимодействует с конструктором, состояние управляется локально
3. **Расчеты в реальном времени**: Статистика, калории и сложность рассчитываются при изменении ингредиентов
4. **Сохранение**: Готовые бургеры сохраняются через POST запрос к `/api/burgers`
5. **Обновление галереи**: React Query автоматически обновляет галерею после сохранения
6. **Feature Flags**: Endpoint версии предоставляет статус feature flags для условного UI

## 🚀 Развертывание {#deployment}

### Процесс сборки

- **Фронтенд**: Vite собирает React приложение в `dist/public`
- **Бэкенд**: esbuild упаковывает серверный код в `dist/index.js`
- **База данных**: Миграции Drizzle применяются через `drizzle-kit push`

### Настройка окружения

```bash
# Обязательные переменные окружения
DATABASE_URL=postgresql://user:password@host:port/database
NODE_ENV=production
```

### Установка и запуск

```bash
# 1. Установка зависимостей
npm install
# 2. Сборка приложения
nJgb
# 3. Применение миграций БД
npm run db:push
# 4. Запуск сервера
npm start
```

### 🎛️ Feature Flags

Приложение поддерживает два feature flag, которые настраиваются через переменные окружения:

1. **`ENABLE_SOCIAL_SHARING`** - Установите в `true`, чтобы включить кнопки поделиться бургером в соцсетях
2. **`ENABLE_RATING_SYSTEM`** - Установите в `true`, чтобы включить систему оценок и комментариев к бургерам

## 🔧 Технологический стек {#tech-stack}

### Основные технологии

- **База данных**: PostgreSQL (настройка через DATABASE_URL)
- **ORM**: Drizzle ORM для типобезопасных операций с БД
- **Клиент БД**: Neon serverless PostgreSQL с пулом соединений
- **UI фреймворк**: React 18 с TypeScript
- **Система сборки**: Vite с кастомной конфигурацией

### UI библиотеки

- **Radix UI**: Headless компоненты-примитивы
- **Tailwind CSS**: Utility-first стилизация
- **Framer Motion**: Библиотека анимаций
- **Lucide React**: Система иконок

### Инструменты разработки

- **TypeScript**: Полная типизация фронтенда и бэкенда
- **ESLint/Prettier**: Качество кода и форматирование
- **Hot Module Replacement**: Сервер разработки с мгновенными обновлениями

## 🔄 CI/CD Pipeline {#cicd}

### Запущенно на

<http://89.169.180.129:5000/>

### DORA-метрики

- Lead Time for Changes: 1 minute 22 seconds
- Time to Restore Service: 13 seconds

Пайплайн на GitLab CI состоит из четырёх стадий:

### Стадии

| Стадия       | Job               | Описание                                                           |
| ------------ | ----------------- | ------------------------------------------------------------------ |
| **build**    | `build_backend`   | Сборка приложения (Vite + esbuild), упаковка артефакта в `.tar.gz` |
| **release**  | `release_backend` | Загрузка артефакта в Nexus repository                              |
| **deploy**   | `deploy`          | Развёртывание на продакшен-сервер по SSH (ручной запуск)           |
| **rollback** | `manual_rollback` | Откат на предыдущую версию (ручной запуск)                         |

### Поток

```
build → release → deploy (manual)
                  ↘ rollback (manual)
```

### Переменные окружения (GitLab CI/CD Variables)

| Переменная              | Описание                                |
| ----------------------- | --------------------------------------- |
| `NEXUS_URL`             | URL Nexus repository                    |
| `NEXUS_REPO`            | Имя репозитория в Nexus                 |
| `NEXUS_LOGIN`           | Логин для Nexus                         |
| `NEXUS_PASS`            | Пароль для Nexus                        |
| `SSH_USER`              | Пользователь на целевом сервере         |
| `SSH_HOST`              | Host продакшен-сервера                  |
| `SSH_PRIVATE_KEY`       | SSH-ключ для подключения                |
| `DATABASE_URL`          | Строка подключения к PostgreSQL         |
| `ENABLE_RATING_SYSTEM`  | Feature flag рейтингов (`true`/`false`) |
| `ENABLE_SOCIAL_SHARING` | Feature flag шаринга (`true`/`false`)   |
